## 3. 3NF / Schema-rule Compliance — ✅ CLEAN

### 3.1 Source-agnostic schema (Rule #1)
**Pass.** Zero `jobber_*`, `airtable_*`, `samsara_*`, `fillout_*`, `odoo_*` columns in business tables. All cross-system identity lives in `entity_source_links`.

### 3.2 Idempotent upserts (Rule #5)
**Pass for runtime paths.** All Edge Function handlers use `findEntityBySourceId` → upsert pattern. `populate.js` has the known caveat (one-shot init, guarded by `assertEmptyOrForced`) — non-issue post-init.

### 3.3 Never hard-delete (Rule #6)
**Pass with one exception.** `dedup_jobber_links.js` does hard-delete the duplicate rows it merges — this is intentional cleanup, not part of the live sync path. `webhook-jobber`'s `PROPERTY_DESTROY` handler hard-deletes properties (leaf table); other DESTROY events use `softStatusFlip` which sets `status='destroyed'` rather than deleting.

### 3.4 Timestamps + money types
All `TIMESTAMPTZ`, all money `NUMERIC(12,2)` per spot-check.

## 7. Code & Scripts — 🟡 SOME CLEANUP

### 7.1 HIGH — Hardcoded token in `airtable_replay.js:22` (FIXED + ROTATED)
```js
const TOKEN = process.env.AIRTABLE_WEBHOOK_TOKEN || '<token-redacted>';
```
Fallback exposed the real token in source. **Fix:** removed fallback (commit `5e00c55`); rotated the value when the repo went public (commit TBD). Old token is dead.

### 7.2 MEDIUM — `dedup_jobber_links.js` DRY-RUN flag honored only for plan, not DELETE
The DRY check at line 81 returns before plan output, but later DELETE statements (lines 135, 139) execute unconditionally if the function is called outside the dry-run early-return. Currently safe because the function returns before reaching the DELETEs in dry mode, but the structure is fragile. Refactor to wrap each side-effect in `if (!DRY)`.

### 7.3 MEDIUM — `test_suite.js` references dropped v1 columns
Queries `samsara_address_id`, `airtable_record_id`, `jobber_client_id` — all dropped in v2. Script will crash if run. **Action:** either rewrite for v2 or delete (it's not invoked anywhere).

### 7.4 LOW — Inconsistent HTTP patterns (3 different helpers across scripts)
`airtable_replay.js` custom helpers, `lib/jobber.js` uses fetch, `lib/sources.js` custom https. Consolidate into shared `lib/http.js`.

### 7.5 LOW — Console.log noise
~419 `console.log` across scripts; no `--verbose` toggle. Add a quiet default with verbose flag.

### 7.6 LOW — `package.json` missing `engines`
CLAUDE.md says Node ≥ 20. Pin in package.json: `"engines": { "node": ">=20.0.0" }`.

### 7.7 LOW — Jobber API version pinned to deprecated `2025-01-20`
Jobber's GraphQL responds with a deprecation warning; current is `2026-04-16`. Bump in `webhook-jobber/index.ts` and `scripts/sync/lib/jobber.js`.
