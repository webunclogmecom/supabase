## How to Restore
If Yan identifies any excluded client that should be in the database, the recovery path is:

1. **Trigger the Jobber webhook manually** for that client — edit the client in Jobber (any touch) and `webhook-jobber` will upsert them into our tables.
2. **Or, one-off GraphQL pull** — query the Jobber API directly for the `client` node by source_id, then insert via the Supabase SQL editor using the same column mapping `webhook-jobber` uses (see `supabase/functions/webhook-jobber/index.ts`).

No data is permanently lost — Jobber is the source of truth. The earlier `scripts/sync/incremental_sync.js` bulk-sync path was removed along with the `daily-sync` GitHub workflow when the architecture pivoted to webhooks-only.
